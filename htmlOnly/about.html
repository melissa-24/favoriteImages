The Idea

As stated on the home page the idea is to build 1 website but do so with different frameworks and languages. The combinations are farm from limitless and as I build this page I have already added more to the original list and I am sure that it might continue to grow.

Part of the other reason is to see not just the different ways that each framework and language is written but also how when all deployed through the same type of hosting account how they each behave. Do they all load the same way? Does 1 version have less code than others? Does one language load faster than another one?

The Breakdown

In the department that I work for we teach a total of 5 stacks...the last being the Capstone part. Every student in our program starts with Web Fundamentals, and then they have the choice of 1, 2, or 3 of the other stacks we teach MERN, Python, and Java. Each of these 3 stacks are Full Stacks meaning the have both a front end and a back end. Where Web Fundamentals is front end only

Here is a rough breakdown of what we teach in each stack

  • Web Fundamentals:
    1. HTML - Got to start here
    2. CSS - Styling it up
    3. JS - Adding functionality to the site
  • Python:
    1. Python Fundamentals
    2. MySQL - The current database structure we teach
    3. Flask - The Framework
  • MERN:
    1. JS Fundamentals
    2. React
    3. Express/Node
    4. MongoDB
  • Java:
    1. Java Fundamentals
    2. Spring

The Combos

As the idea behind this is to create the site in the different languages and frameworks I wanted to take it a little further and make some combos as well.

  • HTML base - This will well this site here but it will have links to the other versions as well as some of the same features.
    • The Nasa API - with out the ability to save the image as there is no backend
    • The Looney Tunes API - again with out the ability to save the image as there is no backend
    • The Data - Each back end will have the ability to return the database data through an API and will be shown here
  • Pug base - This will be similar to the HTML base site to a point
    • This will use Node/Express as a way to display the pug files however like the HTML base will have no physical backend
    • It will have all the same features that the HTML base such as the Nasa API & Looney Tunes API
  • Original Favorite Images App - This is the original version that I built for my python classes and has 1 extra feature that is currently not on the others just to keep the combinations simpler
  • Django Only - This will be build using the Django framework which uses Python and be self contained meaning it is all built with in the framework.
    • This based off of how we teach this at Coding Dojo (or rather as we have moved to Flask)
  • Flask Only - This will be using the Flask framework which also uses Python and like the Django Only version is self contained
    • This is based off of how it is currently taught at Coding Dojo
  • Java Only - This will be created using Spring Boot which is Java based
    • This is based off of how it is currently taught at Coding Dojo
  • MERN - This is a React front end with a MongoDB back end (JavaScript)
    • Like Flask and Java this is also based off of how it is currently taught at Coding Dojo
  • Pug / Express - This will be a pug front end with Node Express backend
  • React / Django - A React front end talking to a Django backend
  • React / Flask - React front end talking to a Django backend
  • React / Express - React front end talking to a Node Express backend
  • React / Java - React front end talking to a Java backend
  • HTML / ??? - HTML front end and unsure what backend (possibly like react and have a version for all backends.
